# pythondata 项目使用说明（飞书版）

领星 + MySQL + 飞书 的数据同步与分析：利润报表、Listing、面料、销量/预估、采购与库存，写飞书多维表或入库。

---

## 一、怎么用（复制即用）

**前提**：项目在服务器路径 `/opt/apps/pythondata`，已配置 `.env`，已建好 venv。

### 1. 用脚本跑（推荐，一条命令一个流程）

在项目根目录执行（或写进 cron）：

| 做什么 | 命令 |
|--------|------|
| 利润报表 + 更新计算字段 | `cd /opt/apps/pythondata && bash scripts/run_profit_report_fee_sync.sh` |
| 数据同步（Listing + 面料 + 下单对比 + 面料预估） | `cd /opt/apps/pythondata && bash scripts/run_data_sync.sh` |
| 采购分析（产品/FBA/库存/销量/采购/库存预估） | `cd /opt/apps/pythondata && bash scripts/run_jobs.sh` |
| 库存同步（库存明细→飞书→货件单号） | `cd /opt/apps/pythondata && bash scripts/run_inventory_sync.sh` |

### 2. 用 Python 单跑某个任务

先 `cd /opt/apps/pythondata` 并激活 venv，再执行：

| 做什么 | 命令 |
|--------|------|
| 拉取利润报表按日数据 | `python -m jobs.Sync_data.fetch_profit_report_msku_daily` |
| 更新利润报表计算字段 | `python -m jobs.Sync_data.update_profit_report_calculated_fields` |
| 创建费用单 | `python -m jobs.Sync_data.create_fee_management` |
| 拉取 Listing | `python -m jobs.Sync_data.fetch_listing` |
| 面料参数（飞书→库） | `python -m jobs.feishu.fetch_fabric_params` |
| 飞书面料数据（飞书→库） | `python -m jobs.feishu.fetch_feishu_data` |
| **销量写飞书（生成销量预估表）** | `python -m jobs.feishu.write_sales_to_feishu` |
| **飞书预计下单→库（收集预估下单表）** | `python -m jobs.feishu.write_order_forecast_to_feishu` |
| 下单对比表 | `python -m jobs.feishu.generate_order_comparison` |
| 面料预估表 | `python -m jobs.feishu.generate_fabric_forecast` |
| 采购分析整条流水线 | `python -m jobs.purchase_analysis.main` |
| 库存明细采集 | `python -m jobs.purchase_analysis.fetch_inventory_details` |
| 库存写飞书 | `python -m jobs.feishu.write_inventory_to_feishu` |
| 货件单号写飞书 | `python jobs/feishu/Shipment_Number.py` |

## 二、这些代码都是什么意思

### 1. 四个主要脚本（.sh）

| 脚本 | 意思 |
|------|------|
| `run_profit_report_fee_sync.sh` | 先拉领星利润报表按日数据，再更新库里计算字段；失败会发飞书。 |
| `run_data_sync.sh` | 顺序做：拉 Listing → 拉面料参数 → 拉飞书面料 → 生成下单对比表 → 生成面料预估表；任一步失败发飞书。 |
| `run_jobs.sh` | 跑采购分析主流程：产品、FBA、库存明细、销量月度、采购单、库存预估。 |
| `run_inventory_sync.sh` | 顺序做：拉库存明细 → 写飞书库存表 → 货件单号写飞书；任一步失败发飞书。 |

### 2. 飞书相关（销量预估表 / 预估下单表）

| 代码 | 意思 |
|------|------|
| `write_sales_to_feishu.py` | 用库里的销量统计 + 库存预估 + 运营预计下单，**生成/更新**飞书「销量预估表」（按店铺，含历史销量、预计销量、预计下单量）。 |
| `write_order_forecast_to_feishu.py` | 从飞书多维表里**把运营填的预计下单量读到库里**，写入「运营预计下单表」。 |

### 3. 领星同步（Sync_data）

| 代码 | 意思 |
|------|------|
| `fetch_profit_report_msku_daily.py` | 领星 → 利润报表按日数据入库。 |
| `update_profit_report_calculated_fields.py` | 用产品、头程等更新利润报表里的计算字段。 |
| `create_fee_management.py` | 按利润报表数据在领星创建费用单。 |
| `fetch_listing.py` | 领星 → 亚马逊 Listing 入库。 |

### 4. 飞书其他

| 代码 | 意思 |
|------|------|
| `fetch_fabric_params.py` | 飞书「定制面料参数」→ 入库。 |
| `fetch_feishu_data.py` | 飞书面料表（款号、季节、面料）→ 入库。 |
| `generate_order_comparison.py` | 采购单 + 运营预计下单 → 生成下单对比表（可写飞书）。 |
| `generate_fabric_forecast.py` | 运营预计下单 + 面料核价 → 面料用量预估。 |
| `write_inventory_to_feishu.py` | 仓库库存明细 → 飞书多维表。 |
| `Shipment_Number.py` | 领星货件 → 按 SPU+箱号聚合 → 飞书货件单号表。 |

### 5. 采购与库存（purchase_analysis）

| 代码 | 意思 |
|------|------|
| `main.py` | 采购分析总入口：产品(7天增量)、FBA、库存明细、销量月度、采购单、库存预估。 |
| `fetch_product.py` | 领星产品管理（7 天增量）。 |
| `fetch_fba_inventory.py` | 领星 FBA 库存。 |
| `fetch_inventory_details.py` | 领星仓库库存明细。 |
| `fetch_sale_stat_v2_msku_monthly.py` | 领星销量统计 MSKU 月度。 |
| `fetch_purchase.py` | 领星采购单。 |
| `generate_inventory_estimate.py` | 生成库存预估表。 |

### 6. 根目录工具脚本

| 文件 | 意思 |
|------|------|
| `retry_failed_fee_orders.py` | 按失败批次重试费用单。 |
| `resubmit_failed_mskus.py` | 对失败 MSKU 逐个重新提交费用单。 |
| `recreate_all_fee_orders.py` | 重建全部费用单。 |
| `search_in_orders.py` | 订单检索。 |

## 三、服务器定时怎么接

- 服务器已有：每天 06:25 执行 `/etc/cron.daily/` 下所有脚本。
- 接法：在 `/etc/cron.daily/` 里加一个无后缀的可执行文件（如 `pythondata-daily`），内容里依次调用上面四个 `run_*.sh` 即可，例如：

```bash
#!/bin/bash
PROJECT_DIR="/opt/apps/pythondata"
"$PROJECT_DIR/scripts/run_profit_report_fee_sync.sh" || true
"$PROJECT_DIR/scripts/run_data_sync.sh" || true
"$PROJECT_DIR/scripts/run_jobs.sh" || true
"$PROJECT_DIR/scripts/run_inventory_sync.sh" || true
```

## 四、环境与维护

- **环境**：Python ≥3.8，依赖见 `requirements.txt`，配置在项目根目录 `.env`（参考 `common/config.py`）。
- **日志**：在项目下 `logs/`，按日期分目录。
- **维护脚本**：`setup_server.sh` 新机部署，`update_project.sh` 拉代码+更新依赖，`update_project_force.sh` 强制与远程一致，`check_status.sh` 查环境/数据库/crontab，`check_venv.sh` 查 venv 与依赖。

---

**说明**：销量预估表由 `write_sales_to_feishu` 写出；运营在飞书表里填的预计下单，用 `write_order_forecast_to_feishu` 收进「运营预计下单表」。这两个目前未放进四个 run_*.sh，需单独跑或自己加进定时。
